# Design System

## Visual Goal

Create a dark, cinematic, mature, editorial interface that supports exploration without looking like a game dashboard.

## Palette Direction

- Near-black
- Warm charcoal
- Soft ivory
- Muted stone
- Controlled amber, wine, or warm metallic accent
- Natural environmental color from imagery

Avoid:

- Bright startup gradients
- Heavy neon
- Excessive purple
- Cartoon game palettes
- Overused glassmorphism

## Typography

Use:

- Refined display typography for titles
- Highly readable sans serif for controls and metadata

## Core Components

- World map
- Environment portal
- Area navigator
- Interactive hotspot
- Object focus panel
- Memory player
- Story card
- Character card
- Continue-exploring card
- Discovery indicator
- Access badge
- Premium lock
- Cinematic video player
- Timeline item
- Ambient audio control
- Map legend
- Breadcrumb or location trail
- Reduced-motion toggle or preference response
- Loading scene
- Error scene

## Hotspot Design

Hotspots should:

- Be discoverable without clutter
- Have accessible labels
- Work with keyboard and touch
- Show focus clearly
- Avoid requiring pixel-perfect interaction
- Indicate new, discovered, and locked states subtly

## Mobile Adaptation

Mobile should not reproduce desktop literally.

Possible mobile pattern:

- Vertical environment scene
- Swipe or scroll between areas
- Tap objects
- Bottom-sheet content reveal
- Compact map
- Clear back-to-world control

## Accessibility

- Captions and transcripts
- Visible focus
- Semantic controls
- Keyboard access
- Reduced motion
- No autoplay sound
- Sufficient contrast
- Text alternative for visual map
- Conventional navigation alternative
