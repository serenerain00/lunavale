# Experience Principles

## 1. The Site Is a World

The user should feel that they entered a place.

Navigation, imagery, sound, transitions, and content structure should reinforce physical and emotional location.

## 2. Exploration Must Produce Meaning

Every discoverable object should lead to something valuable:

- A scene
- A memory
- A journal entry
- A character detail
- A quote
- A production artifact
- An alternate cut
- A premium discovery

Do not fill environments with meaningless hotspots.

## 3. The Camera Is the Interface

Motion should behave like cinematography rather than decorative UI animation.

## 4. Mystery Cannot Replace Usability

Immersive navigation should never trap the user.

Always provide conventional access through:

- Watch
- Stories
- Characters
- World map
- Search
- Account
- Membership

## 5. Locations Carry Emotional State

An environment may have multiple variants:

- Morning
- Night
- Rain
- Summer
- Winter
- Warm
- Empty
- Threatening
- Nostalgic
- Aftermath

The state should connect to a story moment, not function as random decoration.

## 6. Gamification Is Narrative Progress

Useful progression includes:

- Environments visited
- Memories discovered
- Story chapters completed
- Character histories revealed
- Objects examined
- New paths unlocked
- Premium rooms entered

Avoid generic coins, leaderboards, daily streak pressure, or arbitrary XP unless a future product need justifies them.

## 7. Premium Feels Like Deeper Access

Membership should feel like receiving a key to private parts of the world.

## 8. Performance Is Part of the Experience

A cinematic site that loads poorly is not cinematic.

Prioritize:

- Fast initial content
- Progressive enhancement
- Lazy-loaded media
- Mobile behavior
- Motion fallbacks
- Stable layouts
